The OKR System

Objectives & Key Results — a forcing function for honest prioritization across 90-day cycles.

What & Why

OKRs Are Not a To-Do List

OKRs (Objectives & Key Results) were developed at Intel by Andy Grove and popularized at Google. The core insight: most people confuse activity with progress. OKRs force a separation between what you want to achieve (Objective) and how you will measure achievement (Key Results).

At the personal level, OKRs solve a different problem than corporate OKRs. You're not trying to align a team — you're trying to prevent yourself from drifting. A 90-day OKR cycle is the minimum unit of time in which meaningful skill development or project progress is visible. Shorter cycles optimize for activity; longer cycles lose feedback resolution.

The fundamental test: If your OKR score at the end of 90 days is always 1.0 (everything achieved), your Key Results are too easy. If it's always below 0.4, your Objectives are disconnected from your actual constraints. The target zone is 0.6–0.7 — ambitious but achievable with focused effort.

Architecture

Structure of a Personal OKR

OKR Anatomy
Objective = qualitative · inspiring · directional · time-bound to quarter
Key Result = quantitative · binary or measurable · 2–4 per Objective
Score = 0.0 (not started) → 1.0 (fully achieved)
Target = 0.6–0.7 at quarter end (stretch, not guarantee)

# Personal limit: 1–3 Objectives per quarter. More = none done well.
# Total Key Results: 3–8 across all Objectives. Keep it on one page.

Common Failure Modes

What Breaks OKR Systems

Failure Mode Example Fix
Activity KRs "Work on dissertation 4h/day" bad KRs measure outputs, not inputs. "Draft 2,000 words/week" measures output. good
Too many OKRs 7 Objectives, 21 KRs for one quarter Hard limit: 3 Objectives, 8 KRs total. If it doesn't fit, it's not a priority.
Binary KRs "Submit paper" (done or not) Add partial scores: 0.3/0.7/1.0 milestones. Binary KRs give no signal until the last week.
No weekly check-in Set OKRs in January, review in March OKRs must be reviewed weekly (5 min) to function as a navigation system, not a report card.
Sandbagging Consistent 1.0 scores every quarter If you hit 1.0 twice in a row, increase ambition by ~30% next quarter.

Weekly Integration

How OKRs Connect to Weekly Review

During the Friday Weekly Review, the OKR check takes 5 minutes:

Weekly OKR Check (5 min, every Friday)
# For each Key Result, update score estimate
KR score = current progress / target

# Ask the alignment question:
Did this week's work move any KR forward?
→ If yes: continue
→ If no: identify which DW block next week addresses a KR

# Red flag: 2+ consecutive weeks with no KR movement = structural misalignment
